Transaction List: Table vs Cards

On mobile, how should transaction lists (history & pending) display?

A

Compact Table (Horizontal Scroll)

Keep table but hide less important columns on mobile (source, category icon only). Horizontal scroll for full data. Familiar pattern.

┌─────────────────────────┐
│ ☑️ │ 19 Apr │ Gojek    │ → │
├─────────────────────────┤
│   │ 18 Apr │ Starbucks │   │
│   │ 17 Apr │ Tokopedia │   │
└─────────────────────────┘
(→ = scroll right for amount)
B

Stacked Cards (Recommended)

Each transaction becomes a full-width card. Large merchant name, amount, date, category badge — easy to read and tap. Better for small screens.

┌─────────────────────────┐
│ ☑️ 19 Apr               │
│ Gojek                   │
│ Rp 25.000    [Transport]│
├─────────────────────────┤
│ ☑️ 18 Apr               │
│ Starbucks               │
│ Rp 68.000    [Food]     │
└─────────────────────────┘
